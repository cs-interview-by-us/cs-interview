# UDP
### Q1. TCP와 UDP의 차이점
```
TCP는 데이터의 분실, 중복, 순서가 뒤바뀜 등을 자동으로 보정해줘서 송수신 데이터의 정확한 전달을 할 수 있도록 해줍니다. 
-> 수신 여부 파악, 전송 순서 보장, 신뢰성 높음, 속도 느림

하지만 UDP는 분실, 중복, 순서 뒤바뀜 등을 보정해주지 않기 때문에 이를 어플리케이션에서 처리를 해줘야합니다. 
-> 수신 여부 파악 X, 전송 순서 보장 X, 신뢰성 낮음, 속도 빠름

TCP는 1:1 통신이며 UDP는 1:1, 1: N, N:N 통신에 사용이 됩니다. 
```

### Q2. UDP의 사용 예시
```
UDP는 멀티미디어 스트리밍 애플리케이션, 인터넷 전화 애플리케이션과 같은 실시간으로 데이터가 오고가는 서비스에서 많이 사용됩니다. 
또한 온라인 게임에서도 UDP가 더 많이 사용됩니다. 
```

### Q3. 신뢰성을 보장하는 TCP를 쓰지 않고 UDP를 사용하는 이유 
```
TCP 프로토콜의 처리 방식은 높은 신뢰성을 보장하기에, 이에 따라 높은 오버헤드를 가지게 됩니다. 
전송하는 데이터의 헤더 사이즈도 커지고, 만약 전송이 되지 않으면 다시 재전송해야하는 추가적인 동작이 필요합니다.

따라서, UDP 통신 프로토콜은 신뢰성을 보장하지는 않지만 TCP에 비해 빠른 전송 속도를 제공하기에 
✨신뢰성 보장보다 연속성과 성능이 더욱 중요시되는 서비스에서는 UDP가 사용✨됩니다. 

✨UDP 대신 TCP를 사용하게 된다면?
 
📍인터넷 전화: 전송이 실패하면 재전송을 위한 지연 시간 소요됩니다. 
따라서 송신자와 수신자 간의 싱크가 맞지 않게 되어 원활한 상호간의 의사 전달이 불가능합니다. 

📍온라인 게임: 단 1초의 지연 시간이 발생하여도 서비스 이용자는 굉장한 불쾌감을 느끼게 됩니다.
또한 이러한 지연이 점점 커지게 되면 계속해서 다시 렌더링을 해줘야하며 올바른 연결이 이루어지지 않을 수 있습니다.

📍멀티미디어 스트리밍: 싱크가 맞지 않음으로 인해 많은 문제가 발생할 수 있습니다. 
전송이 원활하게 되지 않으면 연속적인 재생 서비스를 제공받지 못할 수 있습니다. 
```
